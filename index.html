Principios de la Teoría del Color

Los pilares que definen cómo percibimos y usamos el color en el diseño de producto.

🔵

Armonía Cromática

Colores que trabajan juntos de forma coherente. Paletas monocromáticas, análogas o complementarias crean experiencias cohesivas.

⚡

Contraste

La diferencia de luminosidad entre fondo y texto. WCAG recomienda un ratio mínimo de 4.5:1 para texto normal y 3:1 para texto grande.

🟢

Psicología del Color

Cada color evoca emociones. El verde = confianza/éxito. El rojo = urgencia/error. El azul = calma/profesional. Úsalos con intención.

👁️

Jerarquía Visual

El color guía la vista del usuario. Los elementos más importantes deben tener colores más saturados o con mayor contraste.

♿

Accesibilidad

El 8% de los hombres tiene algún tipo de daltonismo. No uses el color como único elemento diferenciador de información.

🌡️

Temperatura del Color

Colores cálidos (rojo, naranja) generan urgencia y energía. Colores fríos (azul, verde) generan calma y confianza.

🎯

Regla 60-30-10

60% color dominante (fondos), 30% color secundario (componentes), 10% color de acento (CTAs y elementos de énfasis).

🌑

Modo Oscuro

No es simplemente invertir colores. Usa grises elevados, reduce la saturación y asegura contraste suficiente en todos los estados.

🎚️ Modelos de Color (RGB, HEX, HSL)

HEX: #6496c8
HSL: hsl(210°,48%,59%)

Resumen

Las 7 Reglas de Oro

Principios concretos que puedes aplicar desde hoy en cualquier proyecto de diseño de interfaces.

01

Contraste mínimo 4.5:1

Para texto normal sobre fondo. Usa herramientas como Coolors o el DevTools del navegador.

02

Regla 60-30-10

60% fondo, 30% elementos secundarios, 10% acento. Nunca más de 3 colores principales.

03

Color + texto + ícono

Nunca uses el color como único indicador de estado. Siempre acompáñalo de una señal visual alternativa.

04

Respeta la semántica

Verde = éxito, Rojo = error, Amarillo = advertencia, Azul = información. No rompas estas convenciones.

05

Fondos oscuros: grises elevados

En modo oscuro, usa #121212 como base y eleva con capas de gris. No uses negro puro (#000000).

06

Testea con daltonismo

Usa el simulador de Chrome DevTools (Rendering → Emulate vision deficiencies) antes de publicar.

07

Menos colores, más intención

Una paleta de 2-3 colores usados con propósito siempre supera a una paleta de 8 colores sin sistema.
